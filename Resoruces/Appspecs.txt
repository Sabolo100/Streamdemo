Build a simple but well-structured MVP web app for generating ONE home workout session at a time from a tagged fitness video library.

PROJECT PURPOSE

The app is for non-expert users.
It generates a single workout session from a large video library of home fitness exercises.
This is NOT a multi-week planner.
This is NOT a calendar app.
This is NOT a coach dashboard.
This is NOT a content management system.

It is a simple session-generation tool:
- user provides workout needs and constraints
- app generates one coherent workout session
- session is assembled from the tagged video library
- logic should be deterministic and rule-based
- focus on home-safe, simple, understandable, clean experience

PRODUCT PRINCIPLE

The user should see:
- INPUTS on the left
- GENERATED WORKOUT on the right

The user fills in a few parameters, presses Generate, and receives a structured workout session with ordered blocks:
- Warm-up
- Activation (optional)
- Main block
- Accessory / Core block
- Finisher (optional)
- Cooldown

The generated session must feel logical and professionally structured.

PRIMARY GOAL OF THE MVP

Create a working one-screen MVP web app that:
1. loads workout videos from a local data source derived from the attached taxonomy file
2. lets the user input simple training preferences
3. generates one session using deterministic session-building rules
4. displays the result clearly and understandably
5. prioritizes safe and simple home-appropriate exercise choices

SESSION BUILDER ENGINE

The workout generator must be deterministic and rule-based.
Do not use AI inference during runtime.
Do not call any LLM APIs for session generation.
Use transparent, debuggable utility functions.

The generator should work like this:

1. Read all available videos
2. Filter out incompatible videos
3. Choose a session template based on goal and duration
4. Select candidates for each block
5. Score candidates
6. Build the session block by block
7. Order the exercises logically
8. Assign a simple prescription
9. Render the result

STRICT MVP LOGIC PRINCIPLE

When uncertain, choose:
- safer over more intense
- simpler over more advanced
- clearer over more clever
- deterministic over “smart”
- a coherent basic session over a flashy but unstable session

FILTERING RULES

Before selecting any exercise, filter out incompatible videos.

EXCLUDE VIDEOS THAT:
- require unavailable equipment
- exceed selected impact tolerance
- conflict with selected physical limitations
- are too complex for the selected level
- clearly do not fit the chosen goal
- are not safe enough for a home workout MVP
- are too ambiguous if the metadata is weak

Use conservative filtering.

LEVEL RULES

For beginner:
- exclude complexity = complex
- strongly prefer beginner-friendly videos
- prefer low or medium impact
- prefer simple bodyweight or single-tool exercises
- avoid unstable or advanced combinations
- minimize excessive floor-to-stand switching
- avoid overly technical ballistic exercises

For lower_intermediate:
- allow moderate complexity
- allow some stronger elements if other constraints fit

For intermediate:
- allow most moderate combinations
- still respect safety and limitation filters

For advanced:
- allow the widest range
- but still respect impact and physical limitation constraints

IMPACT RULES

If impact tolerance = low:
- exclude high-impact videos
- strongly prefer low impact
- allow medium only when safe and logical

If impact tolerance = medium:
- allow low and medium freely
- allow some high impact only when goal and level justify it

If impact tolerance = high:
- allow all impact levels except where physical limitations block them

PHYSICAL LIMITATION RULES

If knee_sensitive:
- exclude or strongly downrank jump-heavy elements
- exclude repeated landing patterns
- downrank deep knee-dominant overload if alternatives exist

If lower_back_sensitive:
- exclude or strongly downrank risky ballistic hinges
- exclude high-risk loaded hinge patterns unless clearly safe
- prefer controlled, supported movement

If shoulder_sensitive:
- exclude or strongly downrank overhead pressing
- exclude unstable shoulder-heavy moves
- prefer neutral, controlled upper body options

If wrist_sensitive:
- exclude or strongly downrank long plank-loaded wrist work
- exclude difficult push-up dominant wrist loading if alternatives exist

FOCUS AREA RULES

If focus_area = full_body:
- session should cover more than one major region

If focus_area = lower_body:
- main block should prioritize lower-body patterns but still remain balanced enough to feel coherent

If focus_area = upper_body:
- main block should prioritize upper-body patterns with support work

If focus_area = core:
- core emphasis should be central, but session should still have structure and warm-up/cooldown

BLOCK SELECTION RULES

WARM-UP
Select only videos that are suitable for:
- warm-up
- low intensity
- low fatigue
- low or manageable impact
- preparation, not performance

ACTIVATION
Select only videos suitable for:
- controlled activation
- core / glute / scap prep
- light neural preparation
- non-fatiguing priming

MAIN BLOCK
Select videos that:
- strongly match primary goal
- fit focus area
- are appropriate for level
- fit equipment constraints
- are safe enough for home use
- represent the real training value of the session

ACCESSORY / CORE BLOCK
Select videos that:
- complement the main block
- support stability, core, unilateral control, smaller support patterns
- are not too redundant with previous selections

FINISHER
Optional.
Only include if:
- duration allows it
- goal supports it
- level supports it
- impact tolerance supports it
- the selected exercises are simple and safe under fatigue

COOLDOWN
Select only videos that:
- are cooldown compatible
- are light mobility / stretch / breathing style
- do not produce new fatigue

ORDERING RULES

The final session order must feel logical.

Apply these rules:

1. Warm-up is always first
2. Cooldown is always last
3. Technical or important work comes before simpler fatigue work
4. Do not stack too many similar local stress patterns back-to-back
5. Rotate patterns when possible:
   - squat
   - push
   - hinge
   - pull
   - core
6. Avoid early excessive core fatigue if later trunk stability is needed
7. Avoid sudden high-impact spikes too early
8. Finisher, if present, should use simple and repeatable exercises
9. Avoid chaotic exercise flow
10. Avoid excessive equipment switching
11. Prefer one-tool or bodyweight-dominant combinations
12. Avoid too many floor/standing transitions in beginner sessions

EXAMPLES OF GOOD INTERNAL ORDER
- squat
- push
- hinge
- pull
- core

or
- lower
- upper
- lower
- upper
- core

EXAMPLES OF BAD ORDER
For beginner:
- lunge
- jump lunge
- squat pulse
- wall sit
- jump squat

This overloads one local region with poor sequencing.

SCORING SYSTEM

Implement a readable weighted scoring system to rank candidate exercises.

Use factors like:
- goal match
- focus area match
- level compatibility
- equipment compatibility
- impact compatibility
- physical limitation safety
- session role fit
- beginner friendliness
- redundancy penalty
- diversity bonus
- pattern rotation bonus
- home-safety bonus

Do not use machine learning.
Do not use black-box logic.
Use readable TypeScript utility functions.

PRESCRIPTION RULES

For each selected video, assign a simple, human-readable training prescription.

Supported output styles:
- rep-based
- time-based
- hold-based

Use simple heuristics.

BEGINNER GENERAL FITNESS
- 8–12 reps OR 30–40 sec
- 20–30 sec rest
- 2 rounds

BEGINNER STRENGTH / TONE
- 8–12 reps
- 2–3 rounds
- moderate rest

LOWER INTERMEDIATE
- 10–15 reps OR 40–45 sec
- 15–25 sec rest
- 2–3 rounds

FAT BURN / CONDITIONING
- 35–45 sec work
- 15–20 sec rest
- 2–4 rounds depending on duration

CORE HOLDS
- 20–40 sec
- 2–3 rounds

Do not overcomplicate.
This is an MVP.
Simple and believable is better than over-optimized.
